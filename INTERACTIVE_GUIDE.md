# 🎵 线下互动音乐叠加指南

## 互动流程设计

这个系统设计用于线下互动演出，从最简单的鼓点开始，通过观众的互动请求逐步叠加音轨，最终达到丰富的音乐效果。

### 起始状态
系统默认从 **"🎵 互动起点 - 基础鼓点"** 预设开始，只有一个简单的kick鼓点：
```javascript
$kick: s("bd:1!4")
  .duck("2:3:4").duckattack(.2).duckdepth(.8)
._scope()
```

### 渐进式叠加路径

建议按以下顺序引导观众互动，逐步叠加音轨：

#### 阶段 1: 基础节奏层 (0-5分钟)
**当前状态**: 只有kick
**观众可以提交的请求示例**:
- "加个踩镲" / "加点hi-hat"
- "加个军鼓" / "加snare"
- "节奏再丰富一点"

**预期结果**: 系统会添加 `$chh` (closed hi-hat) 和 `$ohh` (open hi-hat)

#### 阶段 2: 低音层 (5-10分钟)
**当前状态**: kick + hi-hats
**观众可以提交的请求示例**:
- "加个贝斯" / "加低音"
- "需要一些bass"
- "底部太空了"

**预期结果**: 系统会添加 `$bass` (square wave bass)

#### 阶段 3: 主旋律层 (10-15分钟)
**当前状态**: 节奏 + 贝斯
**观众可以提交的请求示例**:
- "加个主旋律" / "加lead"
- "来点trance的感觉"
- "加个可以控制的滤波器"

**预期结果**: 系统会添加 `$lead` 并自动包含 `filter_cutoff` slider

#### 阶段 4: 氛围层 (15-20分钟)
**当前状态**: 节奏 + 贝斯 + 主旋律
**观众可以提交的请求示例**:
- "加个铺底" / "加pad"
- "需要一些氛围感"
- "加点和弦"

**预期结果**: 系统会添加 `$subpad` 和 `$pad`

#### 阶段 5: 装饰层 (20-25分钟)
**当前状态**: 完整的基础音乐
**观众可以提交的请求示例**:
- "加个琶音" / "加arp"
- "来点高音装饰"
- "再华丽一点"

**预期结果**: 系统会添加 `$arp`，达到完整的"User Test Music"效果

### 最终效果
完整叠加后的音轨结构：
```
$kick    - 底鼓 (四拍)
$chh     - 闭镲 (16分音符)
$ohh     - 开镲 (重音)
$bass    - 方波贝斯
$lead    - 主旋律 (supersaw) + filter_cutoff slider
$subpad  - 低频铺底
$pad     - 和弦铺底
$arp     - 琶音装饰
```

## Slider 控制

系统只保留一个slider：**filter_cutoff** (范围 0-8)
- 控制主旋律的滤波器截止频率
- 观众可以通过手机实时控制
- 多人同时控制时会计算合力

## 应急预设

如果互动过程中出现问题，可以通过API快速切换到完整预设：
```bash
# 切换到完整的 User Test Music
curl -X POST http://localhost:3000/api/pattern/1
```

## 技术说明

- AI会根据当前代码状态智能添加音轨，不会重复已有的声部
- 系统使用pull-based同步，确保AI始终基于最新的代码状态
- 每个请求都会经过AST验证，确保代码安全可执行
- 队列系统确保请求按顺序处理，避免冲突
